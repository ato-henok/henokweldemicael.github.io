NASA - Research Internship

I built a Graph Database for NASA's collection of lessons it has learned from different missions and experiments. Graph Databases are used for patterns and relationships recognition. You can do a lot more interesting things with your data if you have Graph Database. For example: Google uses Graph to display relative ads, Facebook implemented its "People You May Know" feature with it, Amazon uses it to recommend you similar items to add, etc. NASA wants to apply Graph Database to its Lessons Learned DB so that we can see patterns from several lessons. Some of these entries might have been submitted by different people or even be from different decades.

  • Company: NASA - Johnson Space Center
  • Date: January to May 2016
  • Type: Software Development

Visual Translation App

Description: Visual Translator is a translation tool that allows you to simply snap a photo of a text and do a 2-way translation between Amharic and English. It uses computer vision to recognize the characters on screen. This project won OSU's 2016 Tech Grant

Design Process: Key element of the user experience in this case is simplicity. The whole home page of the app is a camera. Users can choose the direct of the translation (Amharic -> English or English -> Amharic). As soon as the user snaps a photo, the remaining steps are done automatically. Computer vision kicks in and recognizes the text that's on screen. Then the recognized text is sent to the translation engine.

  • Date: October 2016
  • Type: Computer Vision, App Development, UX/UI Design

AroundUS

Description: AroundUS is an event creating iOS app that puts heavy focus on map. You create one account, but you can either be a host or an event-goer once signed in. You simply open the MAP tab and look at events around you. If you like certain event, you can click on it to go to the event page where you can RSVP. This is a OH/O Hackathon project I built with my friend Mary Alula

Design Process: The user experience of this app incorporates what students usually look for when they want to for to a party or attend an event. The first thing asked if how far the place is, how much it costs to attend, and will there be free food. Events that provide food, have a food icon. We put a big emphasis on map because we want users to simply go "I wonder what's going on around me" and see the events in the area. Finding an event to attend should not be that hard; that's why we made the user experience seamless because we did not like the event-creating features provided by other services.

  • Web: Open on Github
  • Date: November 2015
  • Type: Hackathon Project
