NASA - Research Internship

I built a Graph Database for the collection of data NASA gathers from various missions and experiments it launches. The project was initiated because NASA needed to use Graph Database for its Lessons Learned Database to view and identify patterns within the various lessons compiled from entries submitted by different people and various timelines. Normally, Graph Database is used for pattern and relationships recognition, however there is a lot more you can do with data using it. For instance, Google uses it to display relative ads, and LinkedIn and Facebook implemented their "People You May Know" feature using it, while Amazon uses it to recommend to its customers products similar or complementing to the products they are buying.

  • Company: NASA - Johnson Space Center
  • Date: January to May 2016
  • Type: Software Development

Visual Translation App

Description: This translation tool allows you to simply snap a photo of a text and do a 2-way translation between a non-Latin language and English. In this case, I used Amharic letters. Amharic letters are derived from Ge’ez which is a script also known as Ethiopic, and is used as an alphasyllabary for the Amharic and Tigrinya languages, spoken in Ethiopia and Eritrea respectively. The tool uses computer vision to recognize the characters on screen to create translations. This project won Ohio State University’s 2016 Tech Grant. Full story published in, https://techhub.osu.edu/news/2016/11/16/and-grant-goes-2016-student-project-grant-winners

Design Process: Key element of the user experience in this case is simplicity. The home page of the app is a camera, and users can choose the direction of the translation, whether it is the select language -> English or English -> the select language. As soon as the user snaps a photo, the remaining steps take place automatically. The computer vision kicks in and recognizes the text that is on screen, then the recognized text is sent to the translation engine.

  • Date: October 2016
  • Type: Computer Vision, App Development, UX/UI Design

AroundUS

AroundUS is a OH/IO Hackathon project I built with my friend Mary Alula because we did not like the event-creating features provided by other services and we believed that the user experience when searching for events should be seamless.

Description: AroundUS is an event creating iOS app that puts heavy focus on the use of maps. You would create one account, and you can either be a host or an event-goer once signed in. You simply open the MAP tab and look for events around you. If you like a certain event, you simply click on it to go straight to the event page where you can send an RSVP.

Design Process: The user experience of this app incorporates what students usually look for when they want to go to a party or attend an event. The most important questions that students usually ask is how far the event is, how much it costs, and if there will be free food. Events that provide food, have a food icon. We put a big emphasis on the map because we wanted users to simply think "I wonder what's going on around me" and see the events in the area. Finding an event to attend should not be that hard.

  • Web: Open on Github
  • Date: November 2015
  • Type: Hackathon Project
