NASA - Research Internship

I am building a Graph Database for NASA's collection of lessons it has learned from different missions and experiments. Graph Databases are used for patterns and relationships recognition. You can do a lot more interesting things with your data if you have Graph Database. For example: Google uses Graph to display relative ads, Facebook implemented its "People You May Know" feature with it, Amazon uses it to recommend you similar items to add, etc. NASA wants to apply Graph Database to its Lessons Learned DB so that we can see patterns from several lessons. Some of these entries might have been submitted by different people or even be from different decades.

  • Company: NASA - Johnson Space Center
  • Date: January to May 2016
  • Type: Software Development

JK iOS App

Description: JK is an iOS social app that allows users to share jokes. Every entry goes to the CROWD column. Users can then give Rose if they like it or Tomato if they dislike it. The jokes that received the most Roses on a given day are featured on the STAGE.

Design Process: Key element of the user experience in this case is simplicity, especially in the navigation. Users will are there to view list of jokes, therefore, the core interaction is with the feed. Tabview was used for the different pages. Tabview keeps the navigation simple because all you do is swipe left and right. Even the profile page is in the Tabview and it only consists few menus. As for the backend, Parse was used for both Sata Storage and Push Notifications.

  • App Store: iOS
  • Date: August 2015
  • Type: App Development, UX/UI Design

AroundUS

Description: AroundUS is an event creating iOS app that puts heavy focus on map. You create one account, but you can either be a host or an event-goer once signed in. You simply open the MAP tab and look at events around you. If you like certain event, you can click on it to go to the event page where you can RSVP. This is a OH/O Hackathon project I built with my friend Mary Alula

Design Process: The user experience of this app incorporates what students usually look for when they want to for to a party or attend an event. The first thing asked if how far the place is, how much it costs to attend, and will there be free food. Events that provide food, have a food icon. We put a big emphasis on map because we want users to simply go "I wonder what's going on around me" and see the events in the area. Finding an event to attend should not be that hard; that's why we made the user experience seamless because we did not like the event-creating features provided by other services.

  • Web: Open on Github
  • Date: November 2015
  • Type: Hackathon Project
